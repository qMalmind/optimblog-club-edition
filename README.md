# OptimBlog — версия модуля для Opencart Club

Данный модуль позволяет создавать неограниченное количество статей и категорий для них. Например, вы можете создать категории «Новости», «Статьи», «Акции», а также полноценный блог для вашего магазина с разбивкой материалов по категориям.

## Концепция и отличия от других блогов для Opencart 3

### Другие блоги для Opencart 3

Аналогов нет!  
Большинство блогов и подобных модулей используют концепцию клонирования программного кода (контроллеров и связанных файлов — Model-View-Controller) категорий товаров и самих товаров, превращая скрипты:

1. «Категория товаров» → «Категория статей»  
2. «Товар» → «Статья»  
3. Клонирование таблиц базы данных аналогично категориям и товарам

Такой подход вызывает множество конфликтов. Ошибок и несоответствий на практике более чем достаточно. Описывать их здесь не имеет смысла.

### OptimBlog — концепция и основная идея

1. Категориям присваивается тип — Товар/Статья.  
2. Функционал Статьи расширен аналогично функционалу Товара.  
3. Отзывы аналогично делятся на 2 типа — Товар/Статья. Добавлена возможность отображения «Ответ магазина» на «Отзыв» с использованием HTML.  
4. Этот модуль блога **не заменяет файлы Opencart 3**.  
5. Сохраняется концепция Opencart в названиях и определениях, а также привычный интерфейс для администратора магазина.  
6. Отсутствует конфликт **SEO URL**, так как используется встроенный функционал Opencart, который можно расширять при помощи модификаций.  
7. Модуль на 90% состоит из модификаций, которые можно удалить или отключить в любой момент.

## Функционал и возможности модуля OptimBlog

### Категории:

1. **Заголовок H1**  
2. **Краткое описание** — можно вывести в категории с помощью: `{{ short_description }}`  
3. **Дополнительные изображения** — можно вывести в категории с помощью:  
   `{% for image in images %}` → `{{ image.thumb }}`, `{{ image.popup }}`

### Статьи:

1. **Заголовок H1**  
2. **Краткое описание** — отображается в категориях и модулях статей с помощью: `{{ short_description }}`  
3. **Изображения**  
4. **Дата добавления**  
5. **Дата начала** и **Дата окончания** публикации  
6. **Автор**  
7. **Основная категория** — используется для построения хлебных крошек и формирования Canonical URL для страниц сайта  
8. **Связанные статьи** — невозможна привязка к самой себе. Рекомендации возможны в обе стороны или в одну  
9. **Связанные товары** — также возможны рекомендации в обе стороны или в одну  
10. **Теги**  
11. **Атрибуты**

### Товары:

1. **Заголовок H1**  
2. **Краткое описание** — отображается в категориях и модулях товаров с помощью: `{{ short_description }}`  
3. **Основная категория** — используется для построения хлебных крошек и формирования Canonical URL  
4. **Связанные товары** — невозможна привязка к самому себе. Рекомендации возможны в обе стороны или в одну  
5. **Связанные статьи** — также возможны рекомендации в обе стороны или в одну

### Для разработчиков:

- Некоторые функции и возможности предусмотрены для будущего использования в сторонних модулях. Например, «дополнительные изображения» в настройках категории можно вывести в виде слайдера. А «Производителя» — использовать для привязок и сортировки.
- Разработчики модулей, связанных с отображением товаров, могут легко адаптировать их для Статей, так как PHP-код контроллеров и моделей почти идентичен.
- Используется вёрстка с классами Bootstrap 3, что упрощает адаптацию для любых шаблонов без лишних затрат времени.
